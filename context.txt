Yes — this is a very common situation in voice AI systems.
Sometimes the AI does not have enough information, so it must ask the caller for missing data, then query the client system. This is called slot filling or parameter collection in conversational systems.

You should design your system like a task flow, not a chatbot.

Core Idea
Each action needs required parameters.

Example actions:

get_order_status(order_id)

book_appointment(date, time, patient_name)

check_free_schedule(date)
If some parameter is missing → AI asks the user.

get_order_status(123)

GET /orders/123

1️⃣ Intent Detection
Intent detection means figuring out what the user wants to do.

When a customer speaks, the AI must classify the request into an action.

Example customer speech:

“Where is my order?”

Speech-to-text result:

Where is my order
Intent detection decides:

intent = get_order_status
More Examples
Customer says:

“Book appointment tomorrow”

Intent detected:

intent = book_appointment
Customer says:

“Are you open today?”

Intent detected:

intent = get_business_hours
Implementation
Intent detection can be done by:

LLM classification

2️⃣ Slot Filling
After detecting intent, the system needs parameters.

Example:

get_order_status(order_id)
But the user might not provide the order ID.

User says:

“Where is my order?”

Missing parameter:

order_id
So the AI asks:

“Please tell me your order number.”

User:

“123”

Now the system executes:

get_order_status(123)
Example Appointment Booking
Intent:

book_appointment
Required slots:

name
date
time
Conversation:

User: I want appointment Friday
AI: What time would you prefer?
User: 5 PM
AI: May I know your name?
User: Rahul
Slots collected:

name = Rahul
date = Friday
time = 5 PM
Then execute tool.

Why Slot Filling Is Important
Without slot filling:

AI cannot complete tasks.

It enables structured automation instead of chat.
